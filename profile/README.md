## About CAForge

**CAForge** is a small student-run engineering group. We build projects across a few areas we're actively learning and interested in:

- **AI & Computer Vision** тАФ applied ML, real-time inference experiments
- **Backend Engineering** тАФ API design, service architecture, data pipelines
- **Distributed Systems** тАФ event streaming, caching, multi-service setups
- **Cloud & Infra** тАФ containerization, CI/CD, deployment
- **Full-Stack Apps** тАФ end-to-end projects from data layer to UI

Every project here is built and maintained by the two of us. We don't claim production scale or real-world users тАФ these are learning projects built to a high bar, not commercial products. What we do try to hold ourselves to is writing code that actually runs, documenting it properly, and following the same practices you'd expect in a real engineering team.

## Repository Standards

We aim for every repo in this org to have:

- [x] Dockerized local setup
- [x] A clear README (setup + architecture)
- [x] Documented architecture, with diagrams where it helps
- [x] CI pipeline (lint + test on every PR)
- [x] Tests for the core logic
- [x] License file (MIT by default)
- [x] Conventional Commits
- [x] GitHub Actions for automation

<details>
<summary><b>Why bother with this as students?</b></summary>
<br/>
Because these are the habits that actually matter once you're working on a real team тАФ and it's a lot easier to build them now, on small projects, than to pick them up later under pressure.
</details>
